Instrucciones de despliegue:

1.	Clonar repositorio
2.	Ejecutar script de Base de Datos ubicado en WebApp/DataBase/ProyectoEvaluacionDb.sql
3.	Modificar la cadena de conexión con credenciales de SQLServer en el archivo appsettings.json
4.	Abrir PowerShell para desarrolladores, ubicarse  en la carpeta clonada
5.  Crear esta ruta y dar permisos de administrador C:\inetpub\MyApp
5.	Ejecutar este comandodotnet publish --configuration Release --output "C:\inetpub\MyApp", esto crea el compilado que se coloca en IIS
6.  Verificar que IIS este habilitado, o de lo contrario habilitarlo desde Activar o desactivar características de Windows
7.	Crear un sitio web
	• Nombre: MyApp
	• Puerto: 80 o uno elegido
	• Ruta Física: C:\inetpub\MyApp
	• Validar que el pool de aplicaciones esté configurado para .NET Core
	• Desplegar
	• Navegar a http://localhost:puertoelegido
8. 
Para Habilitar IIS:
Características de Windows:
Marca la casilla que dice “Internet Information Services” y las siguientes:
	Servidor Web (IIS)
	Características comunes de HTTP
	Compatibilidad con ASP.NET
	.NET Extensibility
	CGI
	Autenticación básica o Windows
Application Development Features
	ASP.NET
	.NET Extensibility
	CGI
	ISAPI Extensions e ISAPI Filters
Security
	Autenticación básica o Autenticación de Windows

9. Configurar el web Config, validar que en arguments tenga el mismo nombre del proyecto

<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <system.webServer>
    <handlers>
      <add name="aspNetCore" path="*" verb="*" modules="AspNetCoreModuleV2" resourceType="Unspecified" />
    </handlers>
    <aspNetCore processPath="dotnet" arguments=".\WebApp.dll" stdoutLogEnabled="false" stdoutLogFile=".\logs\stdout" hostingModel="inprocess" />
  </system.webServer>
</configuration>

10. Instalar el .NET Core Hosting Bundle ya que es obligatorio para ASP.NET Core, Elegir .NET 8.0
11. Reiniciar el IIS
12. Validar ruta http://localhost:puertoelegido
